dataAnalyticsModels workflow:
    1. Clean up the example models dbt generated
    2. Configure dbt_project.yml 
    3. Load some of your music app data into RAW schema
    4. Build your first staging model (stg_tracks or stg_playlists)
    5. Build a mart model (something useful like most played artists)
    6. Add dbt tests (not_null, unique etc.)
    7. Then wire it all up with Airflow

Minimal Set You Need For Credibility
    - For your project, you only need:
        - staging models
            - Auth_models:
                - Users
            - Archive_models:
                - Playlist:
                - StreamingLink:
                - Track
        - mart models
            - aggregation:
                - total_users
                - total_playlists
                - total tracks
            - breakdown by playlist type
                - visibility
                - type
            - breakdown per streaming platform
            - breakdown track type
            - analyses of purchase links:
                - Will be good to see how many users are attaching purchase links to their tracks in the playlists.
        - incremental model
        - Basic tests
        - Sources defined
        - Documentation generated
        
Airflow Timing:
    Phase 1 — dbt (now)
    → Load sample data into RAW
    → Build staging + mart models
    → Add tests and documentation

    Phase 2 — Airflow (after dbt is working)
    → Airflow triggers dbt to run on a schedule
    → There's nothing to orchestrate until Phase 1 is done

    Phase 3 — Connect your Django app
    → Airflow extracts data from PostgreSQL
    → Loads into Snowflake RAW
    → dbt transforms it automatically
